요청:


이전 정적 페이지로 이동

자바스크립트가 없던 시절의 브라우저
→ 대부분 각 요청별 특정 경로에 위치한 HTML 파일 불러와 보여주기 수행(라우팅)
→ 여기서, 템플릿 엔진으로 HTML에 데이터 패치해 동적 콘텐츠를 생성(랜더링)하는 방식으로 발전
템플릿 엔진 예: JSP, Thymeleaf, Mustache, Handlebars, Pug, Nunjucks...

* 특징: 데이터 갱신에 페이지 이동(또는 새로고침) 필요(서버 의존도 높음)

MPA(Multi Page Application): 위와 같은 사용자 요청(URL)에 각각 응답하는 HTML 페이지 여럿으로 구성된 웹 애플리케이션
